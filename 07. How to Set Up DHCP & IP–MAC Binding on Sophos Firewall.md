# ✅ How to Set Up DHCP & IP–MAC Binding on Sophos Firewall

***

## 1️⃣ What Is DHCP?

**DHCP (Dynamic Host Configuration Protocol)** is a network service that **automatically assigns IP addresses and other network settings** to client devices.

Without DHCP:

*   Every device must be configured **manually**
*   High chance of IP conflicts
*   Difficult to manage large networks

With DHCP:

*   Devices receive IP automatically
*   Centralized IP management
*   Faster & error‑free configuration

***

## 2️⃣ Advantages of DHCP

✅ **Automatic IP Assignment**  
✅ **Avoids IP Conflicts**  
✅ **Centralized Network Management**  
✅ **Easy for Large Networks**  
✅ **Faster Client Connectivity**  
✅ **Supports IP–MAC binding (static mapping)**

**Real‑World Example:**

> In an office with 50 laptops, DHCP allows all laptops to connect instantly without manual IP configuration.

***

## 3️⃣ Important Notes Before Configuration

*   DHCP works **based on the LAN interface**
*   In this setup:
    *   **Port 08 = LAN**
    *   LAN Network: `192.168.0.0/24`
*   DHCP IP pool must be **within the LAN subnet**
*   Static IP‑MAC bindings must be **outside the DHCP dynamic range**

***

## 4️⃣ How to Configure DHCP on Sophos Firewall

### Path:

    Network → DHCP

Click **Add**

***

### ✅ General Settings

    Name:        Test_DHCP
    Interface:   Port8-LAN-192.168.0.1

***

### ✅ Dynamic Lease IP Range

    Start IP:    192.168.0.220
    End IP:      192.168.0.230
    Subnet Mask: /24 (255.255.255.0)

📌 **Explanation**

*   This range allows **10 devices** to receive DHCP IPs:
        192.168.0.220 – 192.168.0.230
*   Ideal for laptops, mobiles, temporary devices

***

### ✅ DNS Server Configuration

✅ Tick:

    Use device's DNS settings

📌 Sophos will automatically use the DNS servers configured under:

    Network → DNS

Click **Save**

✅ DHCP service starts **immediately**

***

## 6️⃣ What Is IP–MAC Binding?

**IP–MAC Binding** (also called **Static DHCP Mapping**) ensures that a **specific device always gets the same IP address**, based on its **MAC address**.

📌 The IP is still assigned by DHCP, but it never changes.

***

## 7️⃣ Why IP–MAC Binding Is Needed

✅ Printers  
✅ IP Cameras  
✅ Servers  
✅ Network devices  
✅ Devices requiring fixed IP but managed centrally

**Real‑World Example:**

> A printer must always remain on the same IP so users don’t need to update printer settings.

***

## 8️⃣ Where to Configure IP–MAC Binding

### Path:

    Network → DHCP → Static IP MAC Mapping

***

## 9️⃣ Important Rule for IP–MAC Binding

Dynamic DHCP Range:

    192.168.0.220 – 192.168.0.230

🚫 **Do NOT use these IPs for MAC binding**

✅ Use IPs **outside this range**, but within LAN subnet

***

## 🔟 Configure IP–MAC Binding (Example)

### Static IP–MAC Mapping Entry

    Hostname:     7F_Common_Printer
    MAC Address:  a1:b2:c3:d4:e5:f6
    IP Address:   192.168.0.155

Click **Save**

***

## 1️⃣1️⃣ IP–MAC Binding Result

| Device         | MAC Address       | Assigned IP   |
| -------------- | ----------------- | ------------- |
| Common Printer | a1:b2:c3:d4:e5:f6 | 192.168.0.155 |

✅ Whenever the printer reconnects:

*   It always gets **192.168.0.155**
*   No manual IP configuration needed

***

## 1️⃣2️⃣ DHCP vs IP–MAC Binding (Quick Comparison)

| Feature           | DHCP            | IP–MAC Binding    |
| ----------------- | --------------- | ----------------- |
| IP assignment     | Automatic       | Fixed             |
| IP changes        | Yes             | No                |
| Best for          | Users & laptops | Printers, servers |
| Managed centrally | ✅               | ✅                 |

***

## ✅ Best Practices

✅ Keep DHCP range **small & controlled**  
✅ Reserve lower IPs for **servers & printers**  
✅ Document IP–MAC bindings  
✅ Avoid overlapping IP ranges

***

## ✅ One‑Line Summary

> **DHCP automates IP address assignment, while IP–MAC Binding guarantees specific devices always receive the same IP—both managed centrally in Sophos Firewall.**
