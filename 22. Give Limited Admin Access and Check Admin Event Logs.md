# Sophos Firewall Administration Guide

## How to Give Limited Admin Access | How to Check Admin Event Logs

---

# 📘 Overview

This document explains:

1. How to provide **Limited Administrator Access** to a user in the Sophos Firewall.
2. How to check and export **Administrator Event Logs** for auditing and monitoring purposes.

---

# 🔐 Part 1: Give Limited Admin Access

## Step 1: Login to the Sophos Firewall

Login to the Sophos Firewall using your user credentials.

### Access Details

| Field    | Value              |
| -------- | ------------------ |
| URL      | `172.16.16.16:444` |
| Username | `admin`            |
| Password | `*****`            |

### Procedure

1. Open the firewall management portal.
2. Enter the credentials.
3. Click **Login**.

---

## Step 2: Navigate to Device Access Profiles

Go to:

```text
System > Profiles > Device Access
```

Here, you can see the predefined profiles configured by Sophos itself.

---

## Step 3: Create a Custom Permission Profile

To use custom permissions:

1. Click **Add**.

---

## Step 4: Configure the Profile

Fill up the following credentials:

| Field        | Value                  |
| ------------ | ---------------------- |
| Profile Name | `Branch_IT_Team_Admin` |

---

### ⚙️ Configuration Setup

Under the configuration section, configure permissions as per the requirement.

Available permission levels:

* ❌ **None**
* 👁️ **Read**
* ✏️ **Read and Write**

Select the required permissions based on operational necessity.

---

## Step 5: Save the Profile

After the configuration is completed:

1. Click **Save**.

✅ The profile has now been created successfully.

---

## Step 6: Assign Permission to the User

Now we need to provide the permission profile to the user.

Go to:

```text
Configure > Authentication > Users
```

---

## Step 7: Search for the User

Search for the username.

Example:

```text
xitiz.basnet
```

---

## Step 8: Configure User Access

Open the user account and select the following:

| Setting      | Value                  |
| ------------ | ---------------------- |
| User Type    | `Administrator`        |
| Profile Type | `Branch_IT_Team_Admin` |

After configuration:

1. Click **Save**.

✅ Limited Admin Access has now been successfully assigned to the user.

---

# 📊 Part 2: Check Admin Event Logs

## Step 1: Navigate to Compliance Reports

Go to:

```text
Monitor & Analyze > Reports > Compliance
```

---

## Step 2: Configure Report Filters

Fill up the following details:

| Field       | Value                           |
| ----------- | ------------------------------- |
| Show        | `Events, View All`              |
| Choose Date | `From 2026-06-01 to 2026-05-12` |

After entering the required information:

1. Click **Generate**.

---

## Step 3: View Admin Event Logs

You can now see the admin events on the right corner dashboard.

To proceed:

1. Click **View More**
2. Click **OK**

---

## 📋 Available Log Details

The following information can be viewed in the Admin Event Logs:

* 🕒 Time
* 📝 Event Type
* ⚠️ Severity
* 💬 Message
* 👤 User
* 🌐 Source
* ✅ Status

---

## 📤 Export Event Log Details

The event log details can also be exported for:

* Auditing
* Compliance reporting
* Security investigations
* Administrative review

# ✅ Summary

This guide covered:

* Creating a custom limited administrator profile
* Assigning limited admin permissions to users
* Monitoring administrator activities using event logs
* Exporting logs for reporting and compliance purposes

---

# 🛡️ Best Practices

* Grant only the minimum required permissions.
* Regularly review administrator activity logs.
* Export and archive logs periodically for compliance.
* Use custom profiles instead of full administrator access whenever possible.

---

# 📎 Notes

> Ensure that administrative credentials are stored securely and shared only with authorized personnel.

> Review permission assignments periodically to maintain security compliance.
