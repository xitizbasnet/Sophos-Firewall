# 📘 Sophos Firewall – Diagnostics and Troubleshooting Guide

## 📖 Overview

The **Diagnostics** section of Sophos Firewall is one of the most powerful tools available to network administrators. It helps identify, analyze, and troubleshoot network, connectivity, web access, and firewall policy issues.

Whether a user is unable to access a website, internet performance is degraded, or management requests evidence of firewall behavior, the Diagnostics module provides the tools required to investigate and resolve these issues efficiently.

This guide explains:

* 🔍 Why Diagnostics is important
* 🌐 How to troubleshoot website access issues
* 📊 How to monitor firewall and ISP performance
* 🛠 Common diagnostic tools
* 📋 Real-world troubleshooting scenarios

---

# 🎯 Why Diagnostics Is Important

Network administrators frequently encounter situations where users report connectivity or performance problems.

Consider the following real-world scenarios.

---

# 🌐 Scenario 1 – Website Is Not Accessible

A user reports:

> "I am unable to access a specific website."

You perform several checks:

* The website works on a mobile device.
* The website does **not** work from the office network.

At first glance, it may appear that the firewall is blocking the website.

However, even after verifying that:

* The website is already allowed in the Web Policy.
* No firewall configuration has changed.
* The website was working until yesterday.

the issue still exists.

Additionally, multiple users from the same department report the same problem.

## Possible Causes

A website may become inaccessible for several reasons, including:

* DNS resolution problems
* ISP-level blocking
* Government-mandated website restrictions
* Network routing issues
* Security-related ISP filtering during attack events

For example, Internet Service Providers may temporarily block certain categories of websites—such as banking websites—during large-scale cyberattacks or virus outbreaks.

## Troubleshooting Objective

When the website works on a mobile device, one thing becomes clear:

* The website has **not** been blocked by the government in your location.

The next step is to determine whether:

* The ISP is blocking the website.
* DNS resolution is failing.
* The firewall is responsible.

The Diagnostics module provides the information required to identify the actual cause.

---

# 📉 Scenario 2 – Slow Internet Performance

Everything is functioning normally until multiple users suddenly report:

* Internet is extremely slow.
* Emails are not opening.
* Websites take a long time to load.
* Internet connectivity exists but performance is poor.

## Initial Checks

The first step is to examine the firewall diagnostics.

Check:

* ISP bandwidth utilization
* Live traffic graphs
* Current network load

These details are available within the **Diagnostics** section.

---

# 🔥 Firewall Rule Verification

A Sophos Firewall environment may contain:

* 10 firewall rules
* 15 firewall rules
* 20 or more firewall rules

When troubleshooting, you must determine:

* Which firewall rule is processing the user's traffic.
* Which Web Filtering Policy is applied.
* Whether the traffic is being allowed or blocked.

Diagnostics provides complete visibility into these details.

---

# 📑 Providing Evidence to Management

Sometimes users insist that the firewall is blocking legitimate traffic.

Instead of relying on assumptions, Diagnostics allows administrators to present factual evidence by reviewing firewall logs.

These logs can be used to demonstrate:

* Traffic allowed by the firewall
* Traffic blocked by the firewall
* Firewall rule matched
* Destination information
* Policy applied

This information can be presented to:

* IT Managers
* Reporting Managers
* Department Heads
* Senior Management

to confirm whether the firewall is responsible for the issue.

---

# 📰 Scenario 3 – Allowing Website Categories

Different departments often require access to different types of websites.

### Example

The Design Team requires access to image-related websites.

The Accounts Team requests access to news websites.

However, users cannot always provide the exact website URLs.

Instead, they browse various websites, and whenever one is blocked, their work is interrupted.

## Recommended Solution

Instead of allowing individual websites, determine the website's **category** and allow the appropriate category through the Web Policy.

Diagnostics provides the necessary tools to identify website categories.

# 🛠 Tools

The **Tools** section contains multiple utilities used for troubleshooting.

---

## 📋 Log Viewer

The **Log Viewer** is one of the most frequently used diagnostic tools.

It allows administrators to analyze the traffic generated by any device on the network.

### Example

Enter the client IP address:

```text
192.168.2.35
```

The Log Viewer displays complete traffic information.

### Available Information

* ✅ Allowed traffic (Green)
* ❌ Blocked traffic (Red)
* Firewall rule used
* Destination IP address
* Port number
* Protocol

Filtering options are also available to simplify log analysis.

---

## 🧪 Policy Tester

The **Policy Tester** is used to simulate firewall policy decisions.

### Example

A user reports:

> "YouTube is not working."

Instead of testing from the user's computer, use the Policy Tester.

Provide:

* Website
* Source IP Address
* User

The firewall evaluates the request.

### Results

If the Policy Tester reports:

```text
Allowed
```

then the firewall is not causing the issue.

The problem may instead be related to:

* User device
* Antivirus software
* Local operating system
* Browser configuration

---

## 🌐 Ping

Used to verify network connectivity.

Typical use cases include:

* Testing connectivity to remote hosts
* Verifying internet access
* Confirming host availability

---

## 🛣 Traceroute

Traceroute displays the path packets follow from the firewall to the destination.

It is useful for identifying:

* Routing problems
* ISP path issues
* High latency between network hops

---

## 🌍 DNS Lookup

DNS Lookup verifies whether a domain name resolves correctly.

This helps determine whether website access failures are caused by DNS resolution problems.

---

# 📊 System Graph

The **System Graph** is one of the most important monitoring tools in Sophos Firewall.

Whenever users complain about slow internet performance, administrators should begin by reviewing this section.

## Monitor

* Internet bandwidth usage
* ISP utilization
* Firewall CPU usage
* Firewall memory usage

Reports are available for:

* Live usage
* Daily statistics
* Weekly statistics
* Monthly statistics

These graphs help identify abnormal traffic spikes and resource utilization.

---

# 🌐 SD-WAN Performance

The **SD-WAN Performance** section is used to prioritize critical traffic.

### Example

Business applications such as:

* Zoom
* Microsoft Teams
* Google Meet

can be given higher priority than large downloads.

As a result, even if a user downloads large files from Google Drive, voice and video conferencing remain stable.

---

# 🔎 URL Category Lookup

The **URL Category Lookup** tool identifies the web category assigned to a website.

### Example

Suppose the firewall policy allows the **News** category.

A user reports that a news website is still blocked.

After checking the category, you discover that the website is categorized as:

```text
Portal
```

instead of:

```text
News
```

In this case, you must manually allow the appropriate category or create an exception.

---

# 📦 Packet Capture

Packet Capture is an advanced diagnostic utility.

It captures live network packets for detailed protocol analysis.

This feature is primarily used by:

* Advanced Network Administrators
* Sophos Technical Support
* Backend Technical Teams

---

# 🔗 Connection List

The **Connection List** displays active network connections passing through the firewall.

Like Packet Capture, this tool is mainly used during advanced troubleshooting and backend analysis.

---

# ⭐ Most Frequently Used Diagnostic Sections

For day-to-day firewall administration, the two sections most commonly used by network administrators are:

* 🛠 Tools
* 📊 System Graph

These sections provide the majority of the information required for routine troubleshooting.

# 💡 Best Practices

* Always verify whether a website issue is caused by the firewall, DNS, ISP, or external restrictions before modifying firewall policies.
* Use **Log Viewer** to determine whether traffic is allowed or blocked.
* Use **Policy Tester** to simulate firewall decisions before making configuration changes.
* Monitor **System Graphs** whenever users report slow internet performance.
* Verify website categories using **URL Category Lookup** before creating policy exceptions.
* Reserve **Packet Capture** and **Connection List** for advanced troubleshooting scenarios.

# 🎓 Conclusion

The **Diagnostics** section is one of the most valuable components of Sophos Firewall for network troubleshooting and performance analysis.

By understanding and regularly using tools such as **Log Viewer**, **Policy Tester**, **System Graph**, **SD-WAN Performance**, and **URL Category Lookup**, administrators can quickly identify the root cause of network issues, provide evidence-based troubleshooting, and maintain reliable firewall operations.
