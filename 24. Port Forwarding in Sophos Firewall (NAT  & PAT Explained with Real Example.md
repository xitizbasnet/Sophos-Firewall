# 🔐 Port Forwarding in Sophos Firewall (NAT  PAT Explained with Real Example)

---

## 📋 Overview

This document explains how to configure **Port Forwarding (NAT/PAT)** in Sophos Firewall to allow external access to an internal **NVR (Network Video Recorder)** system.

The goal is to enable users outside the office network to securely access CCTV camera feeds using a public IP address.

---

## 🎯 Objective

* Publish an internal NVR to the internet securely.
* Map a public IP and port to an internal private IP and service.
* Allow remote access to CCTV cameras via browser/NVR portal.
* Understand DNAT, Loopback NAT, and Reflexive NAT behavior.

---

## 🏗 Architecture Overview

### 🌐 Network Flow

```
Internet User
     ↓
Public IP: 103.167.99.173:86
     ↓
Sophos Firewall (DNAT Processing)
     ↓
Internal Network
     ↓
NVR Server: 192.168.1.20:86
```

---

### 🔄 NAT Types Used

| NAT Type      | Purpose                                     |
| ------------- | ------------------------------------------- |
| DNAT          | Redirects public request to internal server |
| Loopback NAT  | Allows internal users to access public IP   |
| Reflexive NAT | Handles return traffic correctly            |

## ⚙️ Configuration Steps

---

### 🧩 Step 1: Create Host Object (NVR)

Navigate:

```
System → Hosts and Services → IP Host → Add
```

| Field       | Value        |
| ----------- | ------------ |
| Name        | NVR_01       |
| Description | Office NVR   |
| IP Version  | IPv4         |
| Type        | IP           |
| IP Address  | 192.168.1.20 |

✅ Save configuration

---

### 🔌 Step 2: Create Service (Port Definition)

Navigate:

```
System → Hosts and Services → Services → Add
```

| Field            | Value   |
| ---------------- | ------- |
| Name             | Port_86 |
| Type             | TCP     |
| Source Port      | 1:65535 |
| Destination Port | 86      |

> ⚠️ Note: Destination port must match NVR listening port.

---

### 🛡 Step 3: Create DNAT Rule (Server Access Assistant)

Navigate:

```
Protect → Rules and Policies → Add Firewall Rule → DNAT
```

---

#### 🖥 Internal Server Settings

* IP: `192.168.1.20`
* Create host object: Yes (NVR_01)

---

#### 🌍 Public IP Settings

* Interface: Port2
* Public IP: `103.167.99.173`

---

#### 🔌 Services

* Select: `Port_86`

---

#### 🌐 External Source

* Source: Any

---

#### ✅ Finalize

Click:

```
Save & Finish
```

---

## 🔁 Step 4: Firewall Rule Creation

The system automatically creates:

* DNAT Rule
* Loopback NAT Rule
* Reflexive NAT Rule

Rename for clarity:

```
DNAT to NVR_01 Access
```

---

## 📊 NAT Rules Explained

### 1️⃣ DNAT Rule

```
Public IP → Firewall → Internal NVR IP
```

👉 Converts external request into internal destination mapping.

---

### 2️⃣ Loopback NAT

Allows internal users to access:

```
http://103.167.99.173:86
```

Instead of internal IP.

📌 Use case:

* Head office users
* Branch users

---

### 3️⃣ Reflexive NAT

Ensures return traffic from NVR is correctly routed back through firewall.

📌 Without this:

* Sessions may break
* UI may not load properly

---

## ⚠️ Important Notes

> 🔴 Disabling Reflexive NAT may still allow access, but can cause:

* Session instability
* Video stream interruptions
* Partial UI loading issues

## 🔄 Port Translation Example (PAT Scenario)

### Scenario

| Parameter     | Value        |
| ------------- | ------------ |
| Internal IP   | 192.168.1.20 |
| Internal Port | 86           |
| Public Port   | 80           |

---

### Configuration Change

Edit DNAT rule:

| Field              | Value   |
| ------------------ | ------- |
| Original Service   | Port_86 |
| Translated Service | Port_80 |

---

### Result

```
User hits: 103.167.99.173:86
↓
Redirected to: 192.168.1.20:80
```

## 🔐 Security Considerations

* ❗ Avoid exposing NVR directly to internet without protection
* Use strong admin credentials
* Restrict source IP if possible
* Enable HTTPS if supported
* Apply Geo-IP filtering if available
* Monitor firewall logs regularly

## 🐛 Troubleshooting

| Issue                 | Cause                 | Solution                  |
| --------------------- | --------------------- | ------------------------- |
| Cannot access NVR     | NAT rule missing      | Verify DNAT configuration |
| Page not loading      | Wrong port mapping    | Check service definition  |
| Internal access fails | Loopback NAT disabled | Enable loopback rule      |
| Stream lag            | Bandwidth issue       | Check ISP upload speed    |

---

## 💡 Best Practices

* Always use static IP for NVR
* Document all NAT rules clearly
* Avoid using default ports externally
* Regularly audit firewall rules
* Prefer VPN over direct exposure when possible
