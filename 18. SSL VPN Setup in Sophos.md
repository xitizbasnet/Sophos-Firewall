# Introduction

This document explains how to configure and deploy an SSL VPN using a Sophos Firewall environment. The setup includes:

* Firewall core configuration
* SSL VPN policy configuration
* Authentication setup
* User and group assignment
* VPN client download and installation
* End-user connection process

---

# IPsec VPN vs SSL VPN Comparison

| Feature                         | IPsec VPN                                           | SSL VPN                                        |
| ------------------------------- | --------------------------------------------------- | ---------------------------------------------- |
| **Full Form**                   | Internet Protocol Security                          | Secure Sockets Layer Virtual Private Network   |
| **OSI Layer**                   | Network Layer (Layer 3)                             | Application Layer (Layer 7)                    |
| **Access Level**                | Network-level access                                | Application/User-level access                  |
| **Ports / Protocols**           | UDP 500, UDP 4500, ESP                              | TCP/UDP 443 or custom SSL port                 |
| **Performance**                 | Faster and efficient for site-to-site communication | Slightly slower due to SSL encryption overhead |
| **Reliability on Public Wi-Fi** | Less reliable behind NAT/firewalls                  | More reliable and firewall-friendly            |

# SSL VPN Deployment Overview

SSL VPN can be configured in **three major processes**:

1. 🔧 **Firewall Core Configuration**

   * Device access
   * IP host range
   * Group creation
   * Firewall rules
   * Authentication setup

2. 🌐 **VPN Configuration**

   * SSL VPN global settings
   * SSL VPN policy creation

3. 👤 **User-Level Configuration**

   * VPN portal access
   * Export configuration
   * VPN setup on client computer

# Create VPN IP Host Range

Navigate to:

```text
System > Host and Services > IP Host
```

Click:

```text
Add
```

---

## ⚠️ Important Note

Avoid using commonly used home networks such as:

```text
192.168.0.1
192.168.1.1
```

Also, do not use the same network range already used in the office network.

---

## Configure the Following Settings

| Setting     | Value               |
| ----------- | ------------------- |
| Name        | FOR VPN             |
| Description | FOR VPN             |
| IP Version  | IPv4                |
| Type        | Network             |
| IP Address  | 192.168.99.0        |
| Subnet      | /24 (255.255.255.0) |

Click:

```text
Save > OK
```

# SSL VPN Global Configuration

Navigate to:

```text
Configure > Remote Access VPN > SSL VPN
```

Locate:

```text
SSL VPN Global Settings
```

Open it and configure the following:

| Setting                    | Value                           |
| -------------------------- | ------------------------------- |
| Protocol                   | UDP                             |
| SSL Server Certificate     | Appliance Certificate           |
| Override Hostname          | Leave Empty                     |
| Port                       | 8443                            |
| Assign IPv4 Address        | 192.168.9.0 /24 (255.255.255.0) |
| Assign IPv6 Address        | 2001:dbe::1:0                   |
| Lease Mode                 | IPv4 Only                       |
| IPv4 DNS                   | 8.8.8.8 and 4.4.2.2             |
| Disconnect Dead Peer After | 100                             |
| Disconnect Idle Peer After | 15                              |
| Encryption Algorithm       | AES-128-CBC                     |
| Authentication Algorithm   | SHA2-256                        |
| Key Size                   | 2048 bit                        |
| Key Lifetime               | 28800                           |

# Create SSL VPN Policy

Navigate to:

```text
Configure > Remote Access VPN > SSL VPN > Add > Configure Manually
```

Configure the following:

| Setting                            | Value                                    |
| ---------------------------------- | ---------------------------------------- |
| Name                               | headoffice_SSL_VPN                       |
| Description                        | headoffice_SSL_VPN                       |
| Policy Member                      | test (group generally added here)        |
| Use as Default Gateway             | Enable Radio Button                      |
| Permitted Network Resources (IPv4) | tally_server (choose as per requirement) |
| Permitted Network Resources (IPv6) | Leave Empty                              |
| Disconnect Idle Clients            | Enable Radio Button                      |
| Override Global Timeout            | Leave Empty                              |

> 📝 Default timeout is 15 minutes.

Click:

```text
Apply > OK
```

# Create User Group

Navigate to:

```text
Configure > Authentication > Groups > Add
```

Fill the following details:

| Setting                   | Value                     |
| ------------------------- | ------------------------- |
| Group Name                | headoffice_SSL_VPN_Users  |
| Description               | headoffice_SSL_VPN_Users  |
| Group Type                | Normal                    |
| Surfing Quota             | Unlimited Internet Access |
| Access Time               | Allow All the Time        |
| Network Traffic           | None                      |
| Traffic Shaping           | None                      |
| SSL VPN Policy            | headoffice_SSL_VPN        |
| Clientless SSL VPN Policy | No Policy Applies         |
| L2TP                      | Disabled                  |
| PPTP                      | Disabled                  |
| Quarantine Digest         | Enable                    |
| MAC Binding               | Disabled                  |
| IPsec Remote Access       | Enable                    |
| Sign-in Restriction       | Any Node                  |

Click:

```text
Save
```

# Assign VPN Access to Users

If one of the users wants to work from home and needs VPN access:

Navigate to:

```text
Configure > Authentication > Users > test
```

Under:

```text
Policies
```

Set:

```text
Group: headoffice_SSL_VPN_Users
```

Click:

```text
Save
```

✅ The `test` user can now successfully use the VPN connection.
