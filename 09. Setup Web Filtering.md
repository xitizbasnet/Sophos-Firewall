# 🌐 Web Filtering Configuration Guide  
## Blocking Websites & Categories | Setup Web Filtering

---

## 📌 Overview
This document provides step-by-step instructions to configure web filtering policies, manage website access, and apply rules within the firewall. It also includes best practices for corporate environments.

---

## 🔧 Step 1: Create a Web Policy

**Navigation:**  
`Web > Policies`

### ➕ Create New Policy
- Click **Add Policy**
- Select **Add Web Policy**

### 📝 Policy Details
- **Name:** `HR_Web_Policy`

---

## ✅ Step 2: Configure Allow Rule

### ➕ Add Rule (Allow)
Configure the following:

- **Users:** Anybody  
- **Activities:**
  - Business Cloud Apps  
  - Business Networking  
  - Video Call  
  - Web Mail  
- **Action:**  
  - HTTP → Allow  
  - HTTPS → Allow  
- **Status:** Enabled (🔵 Blue Icon)

---

## 🚫 Step 3: Configure Block Rule

### ➕ Add Rule (Block)
Configure the following:

- **Users:** Anybody  
- **Activities:**
  - Advertisements  
  - Alcohol & Tobacco  
  - Marijuana  
  - Nudity  
  - Pro-Suicide & Self-Harm  
- **Action:**  
  - HTTP → Block  
  - HTTPS → Block  
- **Status:** Enabled (🔵 Blue Icon)

---

## ⚠️ Important Notes
- Rules are processed **top to bottom**.
- ❗ Never place **block rules above allow rules**.
- ✅ Recommended order:
  1. Allow Rule (Top)
  2. Block Rule (Bottom)

---

## 🔍 Step 4: Configure Search Engine Enforcement

- ✅ Enforce Safe Search (✔ Tick Mark)
- ❌ Enforce Additional Image Filtering: Off
- ✅ Enforce YouTube Restriction (✔ Tick Mark)
- **Restriction Level:** Moderate

---

## 💾 Step 5: Save Policy

- Click **Save**

### ⚠️ System Message:
> "Add this policy to a firewall rule for your policy to take effect."

- Close the message for now.

---

## 🔗 Step 6: Link Web Policy to Firewall Rule

### 📍 Current Status
- Policy is **not linked** with any rules.

### ➡️ Navigation:
`Protect > Rules and Policies`

### 🔧 Apply Policy to Existing Rule
Assuming an existing rule:
- **Rule Name:** `HR_Rule_Policy`

#### Steps:
1. Open **HR_Rule_Policy**
2. Go to:
   - `Security Features > Web Filtering > Web Policy`
3. Search and select:
   - `HR_Web_Policy`
4. Add the policy

---

## 🛡️ Step 7: Enable Malware & Content Scanning

Under:  
`Web Filtering > Malware and Content Scanning`

Enable the following:

- ✅ Scan HTTP and Decrypted HTTPS  
- ✅ Use Zero-Day Protection  
- ✅ Scan FTP for Malware  

Click **Save**

---

## 🌍 Step 8: Allow Specific Websites Manually

### 📍 Navigation:
`Protect > Web > Categories`

### ➕ Create Category
- Click **Add**

#### 📝 Configuration:
- **Name:** `Allow_Catg_HR_Policy`  
- **Description:** `Allow_Catg_HR_Policy`  
- **Classification:** Productive  
- **Traffic Shaping Policy:** None  
- **Configure Category:** Local  

#### 🌐 Add Domain/Keyword
- Select: **Import Domain/Keyword → Domain**
- Click ➕ Plus Icon
- Example:
```

[www.firewall.cs](http://www.firewall.cs)

```

Click **Save**

---

## 🔗 Step 9: Add Category to Web Policy

### 📍 Navigation:
`Protect > Web > Policies > HR_Web_Policy`

#### Steps:
1. Go to **Allow Website > Activities**
2. Click **Add New Items**
3. Search and select:
 - `Allow_Catg_HR_Policy`
4. Click **Apply Changes**

---

## ➕ Adding More Websites

- Navigate to:
`Protect > Web > Categories`
- Add new domains under:
**Domain/Keyword**
- Example:
```

[www.nepal.com](http://www.nepal.com)

```
- Click **Save**

---

## 🗑️ Step 10: Delete Web Category

### ⚠️ Common Error:
> "Operation failed. Check if the selected web category is currently in use."

### ✅ Resolution Steps:

1. Go to:
 `Protect > Web`
2. Search and delete:
 - `HR_Web_Policy`

3. Then navigate to:
 `Protect > Web > Categories`
4. Search and delete:
 - `Allow_Catg_HR_Policy`

---

## 🏢 Best Practices for Corporate Environment

### 🔐 Firewall Rule & Web Filtering Strategy

Implement structured policies such as:

- 👔 **Top Management Policy**
- 🏢 **Department-Wise Policy**
- 🌐 **Guest Policy**
- 🧪 **Bypass Policy (Testing Only)**
- Used to test whether a website is blocked by ISP or government
- ⚠️ Must be **disabled after testing**
- 💻 **IT Policy**
- ✅ **Company-Wide Allow Policy**
- 🚫 **Company-Wide Block Policy**
