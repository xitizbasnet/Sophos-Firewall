# ✅ How to Create Firewall Rules & Policies

## + What Is NAT & MASQ (Sophos Firewall)

***

## 1️⃣ What Is a Firewall Rule?

A **firewall rule** defines **how traffic is allowed or blocked between zones** (LAN, WAN, DMZ, VPN).

In Sophos:

*   Traffic is controlled **zone‑to‑zone**
*   Rules are checked **top to bottom**
*   First matched rule decides the action

📌 **No firewall rule = No traffic**

***

## 2️⃣ Why Firewall Rules Are Required

Without firewall rules:

*   LAN users **cannot access the internet**
*   Servers **cannot be published**
*   VPN traffic **will not work**

✅ Firewall rules enforce:

*   **Security**
*   **Access control**
*   **Traffic visibility (logs)**

***

## 3️⃣ Create Firewall Rule: LAN → WAN (Internet Access)

### Navigation

    Rules and Policies → Firewall Rules → Add

***

### ✅ General Settings

    Rule Name:        LAN_to_WAN
    Description:      LAN_to_WAN
    Rule Position:    Bottom
    Action:           Accept
    Rule Group:       None

✅ Tick:

    Log Firewall Traffic

📌 Logging helps in **troubleshooting and monitoring**

***

### ✅ Source Section

    Source Zone:                LAN
    Source Networks & Devices:  Any
    During Scheduled Time:      All the time

📌 This allows **all LAN devices** to access the destination

***

### ✅ Destination & Services

    Destination Zone:   WAN
    Destination Networks: Any
    Services:            Any

📌 This means LAN users can access **any website / any service**

***

✅ Click **Save**

🎉 **Firewall rule is now created**

***

## 4️⃣ Real‑World Scenario: Firewall Rule

> Employees in an office want to browse websites, check email, attend Zoom meetings.

✅ LAN → WAN rule allows:

*   Web browsing (HTTP/HTTPS)
*   Email
*   Software updates
*   Cloud applications

❌ Without this rule → No internet access

***

## 5️⃣ What Is NAT?

**NAT (Network Address Translation)** converts **private IP addresses** into **public IP addresses** (and vice versa).

📌 Private IPs:

*   192.168.x.x
*   10.x.x.x
*   172.16–31.x.x

🌍 Public IP:

*   Provided by ISP (NTC in your case)

***

## 6️⃣ Why NAT Is Needed

Private IPs:

*   Are **not routable on the internet**

Internet:

*   Understands **only public IPs**

✅ NAT allows:

> Internal devices → Internet using a public IP

***

## 7️⃣ Where NAT Is Used

| Scenario                  | NAT Type                 |
| ------------------------- | ------------------------ |
| LAN → Internet            | Source NAT (SNAT / MASQ) |
| Publish internal server   | Destination NAT (DNAT)   |
| Internet access for users | MASQ                     |

***

## 8️⃣ Example: NAT Between LAN & Sophos Public IP

    Laptop (192.168.0.10)
       ↓
    Sophos Firewall
       ↓  (NAT Translation)
    Public IP (103.45.11.154)
       ↓
    Internet

📌 Internet sees only **103.45.11.154**

***

## 9️⃣ Create Linked NAT Rule (LAN → WAN)

When creating the firewall rule, Sophos allows a **linked NAT rule**.

### Steps

✅ Click:

    Create Linked NAT Rule

***

### NAT Rule Configuration

    NAT Rule Name:        LAN_to_WAN
    Translated Source:   MASQ

✅ Click **Save**

***

## 🔟 What Is MASQ (Masquerading)?

**MASQ (Masquerading)** is a type of **Source NAT** where:

*   Sophos automatically replaces **private IP**
*   With **WAN interface public IP**

📌 No need to manually specify public IP

***

## 1️⃣1️⃣ Why MASQ Is Used

✅ Best for:

*   Internet access
*   Dynamic or static public IP
*   Home & office networks

✅ Advantages:

*   Automatic IP translation
*   Easy to configure
*   Most common SNAT method

***

## 1️⃣2️⃣ MASQ Real‑Life Example

| Device  | Private IP   | After MASQ    |
| ------- | ------------ | ------------- |
| Laptop  | 192.168.0.10 | 103.45.11.154 |
| Printer | 192.168.0.20 | 103.45.11.154 |
| Mobile  | 192.168.0.30 | 103.45.11.154 |

🌍 Internet thinks:

> All traffic is coming from **one public IP**

***

## 1️⃣3️⃣ Firewall Rule + NAT (Together)

📌 In Sophos:

*   **Firewall Rule** decides **permission**
*   **NAT Rule** decides **IP translation**

✅ Both are required for LAN → Internet access

***

## 🔁 Traffic Flow Summary

    LAN Device (192.168.x.x)
       ↓
    Firewall Rule (LAN → WAN → ACCEPT)
       ↓
    NAT (MASQ → Public IP)
       ↓
    Internet

***

## ✅ Best Practices

✅ Always:

*   Enable logging initially
*   Use MASQ for internet access
*   Keep NAT linked with firewall rule

❌ Avoid:

*   Any‑Any rules in production
*   NAT without firewall rule
*   Exposing LAN directly to WAN

***

## ✅ One‑Line Final Summary

> **Firewall rules control who can communicate, NAT translates private IPs to public IPs, and MASQ allows LAN users to access the internet using the firewall’s WAN IP.**
