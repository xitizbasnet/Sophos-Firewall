# 📘 Sophos Firewall — Live User Monitoring & VPN Troubleshooting Guide

## 📌 Overview

This document provides a complete guide for monitoring live users, tracking bandwidth usage, disconnecting active sessions, and troubleshooting VPN connectivity issues using the Sophos Firewall dashboard.

The guide is intended for IT administrators and network support teams responsible for managing organizational network access and VPN services.

---

# 🔐 Login to Sophos Firewall

1. Open the Sophos Firewall web portal.
2. Enter your administrator credentials.

```text
Username: admin
Password: *****
```

3. Click **Login**.

---

# 📊 Access Live User Monitoring

Navigate to the following section:

```text
Monitor & Analyze > Current Activities > Live Users
```

This dashboard displays all currently connected users and their live network activities.

---

# 👥 Understanding Concurrent Users

At the top of the dashboard, you can view the total number of active users.

Example:

```text
Concurrent Users: 10
```

This means that **10 users are currently connected to the network**.

---

# 📋 Information Available in Live Users Dashboard

The Live Users dashboard provides detailed information including:

| Parameter             | Description                              |
| --------------------- | ---------------------------------------- |
| 🆔 User ID            | Unique identifier for the connected user |
| 👤 Username           | Authenticated user account               |
| 💻 Client Type        | Connection type (Web, VPN, L2TP, etc.)   |
| 🌐 Host IP            | Assigned IP address                      |
| 🔢 IP Version         | IPv4 or IPv6                             |
| 🖧 MAC Address        | Physical network address                 |
| ⏰ Start Time          | Session start time                       |
| ⬆ Upload              | Upload bandwidth usage                   |
| ⬇ Download            | Download bandwidth usage                 |
| 📡 Data Transfer Rate | Current transfer speed (bits/sec)        |
| 🕒 Internet Usage     | Total session duration                   |
| ⚙ Manage              | Session management options               |

---

# 🔍 Filter Users by Client Type

The dashboard allows filtering users based on connection type, such as:

* Web Client
* VPN
* L2TP
* PPTP
* Other connection methods

This helps administrators analyze specific user sessions more efficiently.

---

# ❌ Disconnect a Single User

To disconnect a single user session:

1. Locate the required **User ID**.
2. Select the checkbox beside the user entry.

Example:

```text
User ID: 220
```

3. Click **Disconnect**.
4. A confirmation popup will appear.
5. Click **Disconnect** again to confirm the action.

---

# ❌ Disconnect Multiple Users

To disconnect multiple users simultaneously:

1. Select the checkbox in the **User ID header**.
2. This will select all visible users in the dashboard.
3. Click **Disconnect**.
4. Confirm the action from the popup window.

---

# 🌐 Live Connections Monitoring

Sophos Firewall also provides a **Live Connections** section for deeper traffic analysis.

Navigate to:

```text
Live Connections
```

This section displays live user connections categorized by:

* 📱 Application
* 👤 Username
* 🌐 Source IP Address

---

# 📈 Monitor High Bandwidth Consumption

If users are consuming unusually high bandwidth:

1. Open the **Live Connections** dashboard.
2. Search using:

   * Username
   * Source IP
   * Application
3. Analyze the bandwidth consumption details.
4. If excessive usage is detected, administrators may disconnect the session if necessary.

This is useful for identifying:

* Heavy downloads
* Streaming activity
* Unauthorized traffic
* Network abuse

# 🛠 VPN Troubleshooting Scenarios

---

# 🏠 Scenario 1 — VPN Not Connecting From Home

## Reported Issue

User reports:

> “My VPN was connecting yesterday, but today it is not connecting.”

---

## Troubleshooting Procedure

Request the user to:

1. Enable a mobile hotspot or mobile data connection.
2. Connect the system using mobile internet.
3. Attempt the VPN connection again.

---

## Expected Result

In most cases, the VPN connects successfully using mobile data.

---

## Root Cause Analysis

This usually indicates that:

* The user's ISP is blocking required VPN services or ports.
* The local home router/firewall is restricting VPN traffic.
* DNS or network filtering is interfering with the VPN connection.

---

## Recommended Solution

Coordinate with:

* The ISP provider
* Local network administrator
* Router configuration team

to allow required VPN services and traffic.

---

# 🏢 Scenario 2 — VPN Suddenly Stops Working in Office

## Reported Issue

User reports:

> “VPN was working in the morning, but suddenly stopped connecting.”

---

## Troubleshooting Procedure

Check the user's authentication settings in Sophos Firewall.

Navigate to:

```text
Sophos > Configure > Authentication > Users
```

Search for the affected username.

Example:

```text
xitiz.basnet
```

---

# 🔑 Check Simultaneous Sign-ins Configuration

Review the setting:

```text
Simultaneous Sign-ins
```

---

## Option 1 — Use Global Setting

If the user is configured with:

```text
Use Global Setting
```

then the firewall applies the globally configured device login limit.

Example:

* One VPN login
* One device only

---

## Option 2 — Custom Device Limit

If a custom value is configured:

```text
2
```

then the user can connect VPN from **2 devices simultaneously**.

If additional access is required:

* Increase the limit:

  * 2 → 3
  * or higher as required

After increasing the allowed device count, the VPN should reconnect successfully.

---

# ⚠ Common Cause of VPN Session Issues

In many situations:

* Firewall rules are correct
* VPN policies are functioning properly
* Authentication settings are valid

However, the issue occurs because of unstable client-side connectivity.

---

# 🌐 VPN Session Timeout Behavior

During VPN usage, the following conditions may occur:

* Internet interruption
* Network instability
* Packet loss
* ISP fluctuation

In these cases:

* The VPN session may remain active on the firewall.
* Timeout sessions may fail to refresh properly.
* Sophos Firewall may still consider the VPN connected.

As a result:

* Users cannot reconnect immediately.
* Simultaneous login limits are reached.
* Authentication conflicts occur.

---

# ✅ Recommended Best Practice

Instead of relying only on:

```text
Global Access
```

use:

```text
Custom Device Access
```

This provides better flexibility for users experiencing unstable internet connections.
