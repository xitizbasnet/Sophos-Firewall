# Overview

When users come to the office and connect to the network, the system should automatically display a login message before allowing internet access. After successful authentication, users can proceed with their daily office tasks.

To make the login page appear automatically for internet access, DNS rules and captive portal settings must be configured properly in the firewall.

---

# What Is User Authentication?

🔐 **User Authentication** is the process of verifying a user's identity before granting access to network resources or the internet.

Instead of allowing anonymous internet access, the firewall identifies users through:

* Username and password
* Active Directory (AD)
* Single Sign-On (SSO)
* Captive Portal login

### Example

A user named `john.doe` connects to the office Wi-Fi. Before internet access is allowed, the firewall asks the user to log in using company credentials.

---

# What Is a Captive Portal?

🌐 A **Captive Portal** is a web page that users must interact with before they can access the internet.

It is commonly used in:

* Offices
* Hotels
* Schools
* Airports
* Public Wi-Fi environments

### Example

When a user connects to the office network and opens a browser, a login page automatically appears asking for authentication credentials.

---

# Why Is User Authentication or Captive Portal Important?

✅ User authentication and captive portals are important because they help organizations:

* Identify who is using the internet
* Monitor user activities
* Apply internet usage policies
* Improve security
* Prevent unauthorized access
* Generate user-based reports
* Control bandwidth usage

### Example

Without authentication, all users appear under one IP address, making it difficult to identify who accessed a specific website.

With a captive portal, each user logs in individually, allowing accurate tracking and monitoring.

---

# What Problem Does a Captive Portal Solve?

🛠️ A captive portal solves several network management and security problems.

### Problems Solved

| Problem                     | Solution Provided by Captive Portal      |
| --------------------------- | ---------------------------------------- |
| Unknown internet users      | Identifies authenticated users           |
| Shared IP addresses         | Maps internet activity to specific users |
| Unauthorized access         | Requires login before internet usage     |
| Difficulty monitoring users | Provides user activity visibility        |
| Lack of accountability      | Tracks individual user sessions          |

### Example

If a user accesses restricted websites, the firewall logs can identify exactly which authenticated user performed the activity.

---

# Network User Tracking Methods

## Static IP Method

One example of tracking users on the internet using a firewall is by assigning **Static IP Addresses**.

### Limitation

This method works only for a limited number of users, such as:

* 70 users
* 100 users

### Problem With Large Networks

What if the number of users increases to around **2000 people**?

Managing static IP addresses manually becomes difficult because:

* IP management becomes complex
* Tracking users becomes inefficient
* Scalability issues arise
* Administrative overhead increases

✅ In large environments, **Captive Portal Authentication** is a better solution.

---

# Scalability Considerations

📈 Captive portals are more scalable for large organizations because they:

* Support thousands of users
* Centralize authentication
* Integrate with Active Directory
* Provide user-based monitoring
* Simplify network management

### Example

A company with 2000 employees can authenticate users through Active Directory instead of manually assigning static IP addresses.

# How to Set Up a Captive Portal

---

## Step 1: Configure DNS Firewall Rule

📍 Navigate to:

```text
Protect > Add Firewall Rule > New Firewall Rule
```

### Firewall Rule Configuration

| Setting              | Value    |
| -------------------- | -------- |
| Rule Name            | DNS_Rule |
| Rule Position        | Top      |
| Action               | Accept   |
| Log Firewall Traffic | Enabled  |

### Source Settings

| Setting                     | Value        |
| --------------------------- | ------------ |
| Source Zone                 | LAN          |
| Source Networks and Devices | Any          |
| During Scheduled Time       | All the Time |

### Destination Settings

| Setting              | Value |
| -------------------- | ----- |
| Destination Zone     | WAN   |
| Destination Networks | Any   |
| Services             | DNS   |

### NAT Rule Configuration

Enable:

```text
Create Linked NAT Rule
```

Then configure the following:

| Setting                       | Value    |
| ----------------------------- | -------- |
| Rule Name                     | DNS_Rule |
| Rule Position                 | Top      |
| Original Source               | Any      |
| Original Destination          | Any      |
| Original Service              | Any      |
| Translated Source (SNAT)      | MASQ     |
| Translated Destination (DNAT) | Original |
| Translated Service (PAT)      | Original |

Click:

```text
Save
```

---

## Important Note

> ⚠️ **Sophos Captive Portal Default Port Number:** `8090`

---

## Step 2: Configure Web Authentication

📍 Navigate to:

```text
Configure > Authentication > Web Authentication
```

---

### If Active Directory (AD) SSO Is Configured

Enable the following radio button:

```text
Kerberos & NTLM
```

---

### If AD SSO Is NOT Configured

When an unauthenticated user reaches a block page:

✅ Tick mark on:

```text
Show Captive Portal Link
```

---

## Captive Portal Behavior Settings

Enable the following options:

### General Settings

✅ Tick mark on:

```text
Show User Portal Link
```

✅ Tick mark on:

```text
Show Web Page After Sign-In
```

---

### Open Web Page

Enable radio button:

```text
In Captive Portal Window
```

---

### Web Page

Enable radio button on:

```text
Originally Requested by User
```

---

### Sign Out User

Enable radio button:

```text
Never
```

---

### HTTP Redirection

✅ Tick mark on:

```text
Use Insecure HTTP Instead of HTTPS
```

Click:

```text
Apply
```

---

## Confirmation Prompt

The system will display the following message:

```text
Are you sure you want to update the captive portal setting?
```

Click:

```text
OK
```

✅ The Captive Portal page is now ready to launch.

# Step 3: Link Captive Portal With Firewall Policies

📍 Navigate to:

```text
Protect > Rules and Policies
```

---

## Example Policy

Suppose there is a policy named:

```text
HR_Rule_Policy
```

### Edit the Policy

Enable:

```text
Match Known Users
```

### User or Groups

Select:

```text
HR_Group
```

Click:

```text
Save
```

# How to Manage User Login Limits

👥 User login limits can be managed through:

* Firewall authentication policies
* Active Directory group policies
* Session timeout settings
* Concurrent login restrictions

### Example

An organization may allow:

* One active session per user
* Limited login duration
* Department-based internet access policies

# How to Manually Disconnect Users

📍 Navigate to:

```text
Monitor & Analyze > Current Activities > Live Users
```

---

## Disconnecting a User

1. Select the desired user
   Example:

```text
xitiz.basnet
```

2. Click:

```text
Disconnect
```

3. Confirm disconnect action.

✅ The user will be disconnected from the captive portal session.
