# ✅ How to Create Users & User Groups in Sophos Firewall

***

## 1️⃣ What Is a User in Sophos Firewall?

A **User** in Sophos Firewall represents a **person or identity** who accesses the network or internet.

Users are used for:

*   User‑based firewall rules
*   Web & application filtering
*   Bandwidth control
*   Access time restrictions
*   Reporting & monitoring

📌 **User‑based security is more powerful than IP‑based security**, because IPs can change but users don’t.

***

## 2️⃣ Why Do We Create Users?

✅ Apply security policies per person  
✅ Track internet usage by user  
✅ Restrict access time & bandwidth  
✅ Assign department‑wise rules (HR, IT, Finance)

**Real‑World Example:**

> IT users may have unlimited internet, while HR users may have restricted access.

***

## 3️⃣ Create a User in Sophos Firewall

### Navigation Path

    Configure → Authentication → Users

Click **Add**

***

### ✅ User Configuration Details

    Username:       xitiz.basnet
    Name:           xitiz.basnet
    User Type:      User
    Profile:        Profile
    Password:       ********
    Confirm Password: ********
    Email:          xitizbasnet@abc.com

***

### ✅ Policies Section

    Group:          Open Group
    Surfing Quota:  Unlimited Internet Access
    Access Time:    Allowed All the Time
    Network Traffic: None
    Traffic Shaping: None

📌 At this stage, the user is **not restricted**.

***

### ✅ Other Settings

**Simultaneous Sign‑ins**

    Unlimited (Recommended: 1)

📌 Allowing multiple logins (e.g., mobile + laptop) is **not recommended** for security reasons.

✅ Keep it restricted in real environments.

***

Click **Save**

✅ **User is successfully created**

***

## 4️⃣ What Is a User Group in Sophos Firewall?

A **User Group** is a **collection of users** that allows you to apply **common policies** to multiple users at once.

Instead of configuring policies for each user:
✅ Apply one policy to a group
✅ Add/remove users from group as needed

***

## 5️⃣ Why User Groups Are Important

✅ Easier management  
✅ Department‑wise security  
✅ Scalable for large organizations

**Real‑World Example:**

*   IT Department → Full access
*   HR Department → Limited access
*   Guest Users → Internet only

***

## 6️⃣ Create a User Group

### Navigation Path

    Configure → Authentication → Groups

Click **Add**

***

### ✅ Group Configuration

    Group Name:     IT Department
    Group Type:     Normal

***

### ✅ Group Policies

    Surfing Quota:   1 Month Internet Access
    Access Time:     Allowed All the Time
    Network Traffic: 100 MB Total Data Transfer
    Traffic Shaping: Extremely Limited User

📌 These policies will apply to **all users in the IT Department group**.

Click **Save**

✅ **Group created successfully**

***

## 7️⃣ Move a User into a Group

Now we assign user **xitiz.basnet** to **IT Department**.

### Steps

Go to:

    Configure → Authentication → Users

Search for:

    xitiz.basnet

Double‑click to **Edit**

***

### ✅ Update Group Membership

    Group: IT Department

Click **Save**

✅ User is now part of **IT Department**

***

## 8️⃣ What Happens After Adding User to Group?

📌 Group policies **override individual user settings**

### Effective Policy for xitiz.basnet:

*   Internet limited to **100 MB**
*   Extremely limited bandwidth
*   Access allowed all day
*   Monitored & logged

***

## 9️⃣ User vs Group (Quick Comparison)

| Feature       | User              | Group             |
| ------------- | ----------------- | ----------------- |
| Identity      | Individual person | Department / Team |
| Policy scope  | Single user       | Multiple users    |
| Scalability   | Low               | High              |
| Best practice | Minimal use       | ✅ Recommended     |

***

## ✅ Best Practices (Enterprise Standard)

✅ Use **groups instead of individual policies**  
✅ Keep simultaneous login to **1 per user**  
✅ Create department‑wise groups  
✅ Apply firewall rules based on **user groups**  
✅ Enable logging for audit & compliance

❌ Avoid:

*   Managing policies per user
*   Unlimited sign‑ins
*   Using users without groups in large networks

***

## ✅ One‑Line Summary

> **Users represent individual identities, groups apply shared policies, and together they enable user‑based security and access control in Sophos Firewall.**
