Sophos Firewall supports two types of backup:

2. **Automatic Scheduled Backup**

---

# 🛠️ Manual Backup Procedure

## Step 1: Login to Sophos Firewall

Login to Sophos Firewall using the appropriate administrator credentials.

---

## Step 2: Navigate to Backup & Restore

Go to:

```text
System > Backup & Firmware > Backup & Restore
```

---

## Step 3: Fill Up the Backup Settings

Configure the following settings:

| Setting             | Value                        |
| ------------------- | ---------------------------- |
| Backup Mode         | Local                        |
| Backup Prefix       | HO_Sophos_Backup_May_09_2026 |
| Frequency           | Daily                        |
| Schedule            | 09:00                        |
| Encryption Password | *********                    |

---

## Step 4: Start Manual Backup

Click:

```text
Backup Now
```

---

## 📝 Note

### Before Clicking "Backup Now"

```text
Last backup taken on Tue 24 March 2026 09:00:09
```

### After Clicking "Backup Now"

```text
Last backup taken on Saturday May 09, 2026 09:00:09
```

---

## Step 5: Download the Backup File

Click on the **Download Icon** to download the backup file.

Enable the following option:

```text
☑ Download Encrypted Backup
```

Then click:

```text
Download Backup
```

---

## ✅ Result

The backup file will now be successfully downloaded.

---

# 🔒 Why Encryption Password Is Required

The encryption password protects the backup configuration file from unauthorized access.

Without encryption, anyone with access to the backup file could:

* View firewall settings
* Access VPN configurations
* See network information
* Restore configurations on another device

---

## 💡 Example Scenario

Suppose an employee downloads the firewall backup file and stores it on a USB drive.

If the USB drive is lost or stolen:

* Without encryption → Anyone can open and misuse the firewall configuration.
* With encryption → The backup file cannot be restored or accessed without the encryption password.

This adds an important layer of security to the backup process.

# 📧 Configure Email Settings for Automatic Backup

Before configuring automatic scheduled backups, email settings must be enabled in Sophos Firewall.

---

## Step 1: Navigate to Notification Settings

Go to:

```text
System > Administration > Notification Settings
```

---

## 📝 Note

The email used in this example is hosted on Google Email (Gmail).

---

## Step 2: Configure Email Settings

Fill in the following details:

| Setting                         | Value                                       |
| ------------------------------- | ------------------------------------------- |
| Send Notification Via           | External Email Server                       |
| Mail Server IPv4 Address / FQDN | smtp.gmail.com                              |
| Port                            | 465                                         |
| Authentication                  | Basic                                       |
| Username                        | [backup@gmail.com](mailto:backup@gmail.com) |
| Password                        | ********                                    |
| Connection Security             | SSL/TLS                                     |

---

## ⚠️ Important Note

The password used here is **NOT** the normal Gmail login password.

It must be generated using a **Google App Password**.

---

## Step 3: Configure Email Addresses

| Setting                            | Value                                         |
| ---------------------------------- | --------------------------------------------- |
| From Email Address                 | [backup2@gmail.com](mailto:backup2@gmail.com) |
| Send Notification To Email Address | [backup@gmail.com](mailto:backup@gmail.com)   |
| Management Interface IP Address    | port1 - 172.16.16.16                          |

---

## Step 4: Test Email Configuration

Click:

```text
Test Mail > Send
```

After successful testing:

```text
Apply > OK
```

## Step 1: Login to Gmail

Login to Gmail using your account credentials.

---

Navigate through the following path:


# ⚠️ Consequences of Restoring Firmware in Sophos

Restoring firmware or configuration backups may cause several operational impacts.

---

## Possible Consequences

### 🔄 Work Needs to Be Done Again

Some configurations may require reconfiguration after restoration.

Example:

* VPN tunnels may need to reconnect
* User sessions may terminate
* Policies may require validation

---

### ⏱️ Downtime

During restoration:

* Firewall services may restart
* Internet connectivity may temporarily stop
* Users may lose network access

---

### 🧩 Firmware Version Must Be Checked

Before restoring a backup:

* Ensure the firmware version is compatible
* Different firmware versions may cause restoration issues

---

## 💡 Real Scenario Example

A company restored a backup from an older firmware version into a newer Sophos Firewall firmware.

Result:

* Some firewall rules failed
* VPN services stopped working
* Administrators had to manually recreate several configurations

This caused unexpected downtime during office hours.

# 🛠️ Firmware Upgrade Procedure

## Step 1: Navigate to Firmware Section

Go to:

```text
System > Backup & Firmware > Firmware
```

---

## Step 2: Check for New Firmware

Under the section:

```text
Latest Available Firmware
```

Click:

```text
Check for New Firmware
```

---

## Step 3: Download Firmware

Under:

```text
Action
```

Click:

```text
Download
```

---

## Step 4: Install Firmware

After downloading, the following option will appear:

```text
Install
```

Click:

```text
Install
```

to proceed.

---

## Step 5: Wait for Completion

Wait until the firmware installation process finishes completely.

The firewall may reboot during this process.

# ✅ Best Practices for Firmware Upgrade

## Before Upgrade

* Take a full backup first
* Verify internet stability
* Check firmware compatibility
* Read release notes
* Schedule maintenance window

---

## During Upgrade

* Do not power off the firewall
* Monitor installation progress
* Ensure users are informed about downtime

---

## After Upgrade

* Verify internet connectivity
* Test VPN connections
* Check firewall rules
* Validate services and policies
* Confirm firmware version updated successfully
