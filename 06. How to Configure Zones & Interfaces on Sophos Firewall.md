# ✅ How to Configure Zones & Interfaces on Sophos Firewall

### (LAN | WAN | DMZ with Real‑World Scenarios)

***

## 1️⃣ What Are Zones in Sophos Firewall?

A **Zone** is a **logical security boundary** used by Sophos Firewall to **control traffic flow and apply security rules**.

### Common Zones Available by Default

*   **LAN** – Trusted internal network
*   **WAN** – Untrusted internet network
*   **DMZ** – Semi‑trusted public server network
*   **VPN** – Remote and site‑to‑site VPN traffic
*   **WiFi** – Wireless clients

📌 **Firewall rules are always created between zones**, not just IPs.

***

## 2️⃣ Zone Explanation with Real‑World Scenarios

### 🔵 LAN (Local Area Network)

**Purpose:** Trusted internal users and devices.

**Examples:**

*   Office desktops & laptops
*   Printers, IP cameras
*   Internal servers (File server, AD)

✅ **Real Scenario**

> Employees browsing internet, accessing internal file server, printing documents.

**Allowed Traffic:**

*   LAN → WAN ✅
*   LAN → LAN ✅
*   WAN → LAN ❌ (by default)

***

### 🔴 WAN (Wide Area Network)

**Purpose:** Internet / ISP network.

**Examples:**

*   ISP modem connection
*   Public IP address
*   External users

⚠️ Considered **untrusted**.

✅ **Real Scenario**

> NTC ISP provides a public IP to connect the office to the internet.

**Allowed Traffic:**

*   WAN → LAN ❌ (blocked unless explicitly allowed)
*   LAN → WAN ✅

***

### 🟡 DMZ (Demilitarized Zone)

**Purpose:** Host public‑facing servers securely.

**Examples:**

*   Web server
*   Mail server
*   FTP server

✅ **Real Scenario**

> Company website hosted internally but accessible from the internet without exposing LAN.

**Typical Traffic:**

*   WAN → DMZ ✅ (limited ports only)
*   DMZ → LAN ❌
*   LAN → DMZ ✅ (admin access)

📌 **DMZ must use a different subnet than LAN & WAN**

***

### 🟣 VPN

**Purpose:** Secure remote or branch connectivity.

✅ **Real Scenario**

> Employees working from home access office applications securely.

***

### 🔵 WiFi

**Purpose:** Wireless users (employees / guests).

✅ **Real Scenario**

> Guest WiFi allowed only to internet, blocked from LAN.

***

## 3️⃣ Configure WAN Interface (Port 7 – NTC ISP)

### Steps

**Go to:**  
`Network → Interfaces`

### Select **Port 7** and configure:

    Name:          NTC_ISP
    Hardware:      Port 7
    Network Zone:  WAN

### IPv4 Configuration

✅ Enable IPv4

    IP Assignment: Static
    IPv4 Address: 103.45.11.154/24

### Gateway

    Gateway Name: NTC_ISP
    Gateway IP:   103.45.11.150

❌ Do NOT enable IPv6

***

### Advanced Settings

**Port Settings**

*   Link Mode: Automatic
*   Auto‑Negotiation: Enabled
*   FEC: Off

**Interface Settings**

*   MTU: 1500
*   Override MSS: Disabled
*   Use Default MAC Address ✅

✅ **Save**

📌 Now connect the NTC ISP Ethernet cable to **Port 7**

***

### Verify Internet Connectivity

Go to:

    Network → WAN Link Manager

✅ **Green Status** → Internet is working  
❌ **Red Status** → ISP issue

***

## 4️⃣ Configure LAN Interface (Port 8 – Internal Network)

**Go to:**  
`Network → Interfaces → Port 8`

    Name:          LAN_08
    Hardware:      Port 8
    Network Zone:  LAN

### IPv4 Configuration

    IP Assignment: Static
    IPv4 Address: 192.168.1.1/24

### Gateway

    Gateway Name: Empty
    Gateway IP:   Empty

📌 LAN interface never uses a gateway

❌ Do NOT enable IPv6

***

### Advanced Settings

*   MTU: 1500
*   Override MSS: Disabled
*   Default MAC: Enabled

✅ **Save**

📌 Connect **Port 8 → Switch**

✅ All devices connected to the switch (PCs, printers, cameras) will now access the internet once firewall rules are in place.

***

## 5️⃣ Configure DMZ Interface (Port 6 – Public Servers)

**Go to:**  
`Network → Interfaces → Port 6`

    Name:          DMZ_06
    Hardware:      Port 6
    Network Zone:  DMZ

### IPv4 Configuration

    IP Assignment: Static
    IPv4 Address: 183.178.1.1/24

### Gateway

    Gateway Name: Empty
    Gateway IP:   Empty

❌ Do NOT enable IPv6

✅ **Save**

⚠️ **Important Note**

> DMZ subnet must never match LAN or WAN subnet.

***

### Real DMZ Example

| Server Type | IP           |
| ----------- | ------------ |
| Web Server  | 183.178.1.10 |
| Mail Server | 183.178.1.20 |

Firewall rules will allow:

*   WAN → DMZ (80, 443)
*   LAN → DMZ (Admin access)

    WAN  → DMZ   ✅ Limited access
    LAN  → DMZ   ✅ Allowed
    DMZ  → LAN   ❌ Blocked
