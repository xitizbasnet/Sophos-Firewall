# 📘 Sophos Firewall – Command Line Interface (CLI) Guide

## 📖 Overview

The **Command Line Interface (CLI)** is a powerful feature of Sophos Firewall that is often overlooked by beginners. While the **Graphical User Interface (GUI)** is ideal for everyday administration and routine troubleshooting, the CLI provides a faster and more efficient environment for **advanced configuration, monitoring, and troubleshooting**.

In many advanced support cases, the Sophos Technical Support team may request CLI access to diagnose and resolve complex issues.

This guide explains:

* 🔹 How to access the Sophos Firewall CLI
* 🔹 Basic CLI navigation
* 🔹 Common troubleshooting commands
* 🔹 Monitoring and diagnostic commands
* 🔹 Important menu options available within the CLI

After completing this guide, you will have a solid understanding of how to confidently navigate and use the Sophos Firewall CLI.

---

# 🎯 GUI vs CLI

For day-to-day firewall management, administrators typically use the **GUI** by:

1. Opening a web browser
2. Entering the firewall IP address
3. Logging in using the administrator credentials

While this method is sufficient for most administrative tasks, the **CLI** becomes extremely useful when:

* Performing advanced troubleshooting
* Running diagnostic commands
* Monitoring firewall processes
* Working with Sophos Technical Support
* Accessing features not available in the GUI

---

# 🔐 Methods to Access the Sophos Firewall CLI

There are **three supported methods** to access the Sophos Firewall Command Line Interface.

---

## Method 1 – Access Using PuTTY

PuTTY is one of the most common SSH clients used by network administrators.

### Steps

1. Open **PuTTY**.
2. Enter the **Firewall IP Address**.
3. Click **Open**.
4. When prompted, enter:

   * **Username:** Default administrator account (`admin`)
   * **Password:** Administrator password
5. The Sophos Firewall CLI menu will appear.

The CLI opens as a **menu-driven interface**, displaying numbered options.

Example information displayed includes:

* Firewall Version
* Firewall Model
* Hostname

The interface prompts you to choose menu options by entering numbers from **0–7**.

---

## Method 2 – SSH from Windows CMD or Linux Terminal

Instead of using PuTTY, you can connect directly through the operating system.

### Windows

Open **Command Prompt (CMD)**.

### Linux

Open **Terminal**.

Run the following command:

```bash
ssh admin@<firewall IP>
```

After pressing **Enter**, the system prompts for the administrator password.

Once authenticated, the same CLI menu appears.

---

## Method 3 – Access Through the Firewall GUI

The CLI can also be opened directly from the Sophos Firewall web interface.

### Steps

Navigate to:

```text
Control Center
└── Top-right Corner
      └── Console
```

Select **Console**, enter the administrator password, and the CLI opens.

---

# 📌 Summary of Access Methods

| Method                       | Description                                             |
| ---------------------------- | ------------------------------------------------------- |
| 🖥 PuTTY                     | SSH connection using PuTTY                              |
| 💻 Command Prompt / Terminal | SSH using native operating system tools                 |
| 🌐 Firewall GUI              | Built-in Console from the Sophos Firewall web interface |

All three methods provide access to the same CLI environment.

# 🧭 CLI Main Menu

The Sophos Firewall CLI presents a menu-driven interface with options numbered **0–7**.

> **Note:**
> It is not practical to cover every submenu and configuration option in a single guide. This document provides an overview of the available sections and their primary functions.

---

# 1️⃣ Network Configuration

Navigate to:

```text
1 → Network Configuration
```

This section allows administrators to view and modify network settings.

## Available Tasks

* View configured interfaces
* Configure IP addresses
* Modify subnet masks
* Configure interface settings

### Example

Choose:

```text
1 → Network Configuration
1 → Interfaces
```

The CLI displays all configured interfaces.

After reviewing the interface details, the system asks whether you want to make changes.

If **Yes** is selected, you can modify:

* IPv4 Address
* Subnet Mask
* Other interface parameters

If no changes are made, the CLI displays a message similar to:

```text
IP address not changed.
```

DNS settings can also be configured from the Network Configuration menu.

To return to the previous menu:

```text
Press 0
```

---

# 2️⃣ System Configuration

## Available Options

# 4️⃣ Device Console

The **Device Console** is one of the most frequently used sections within the Sophos CLI.

Most troubleshooting and diagnostic work is performed here.

---

## 💡 Important Tip

There is **no need to memorize CLI commands**.

Simply press the **TAB** key **twice**.

The CLI automatically displays all available commands, including:

* clear
* ping
* telnet
* ssh
* show
* drop-packet

and many others.

---

# 📡 Ping Command

To verify network connectivity:

```bash
ping google.com
```

The firewall continuously sends ICMP echo requests.

Stop the process using:

```text
Ctrl + C
```

---

# 🌍 Traceroute Command

Run:

```bash
traceroute google.com
```

The CLI also supports **TAB completion**, allowing commands to auto-complete.

The traceroute displays the path packets take across the network.

Stop execution with:

```text
Ctrl + C
```

---

# 📊 Show Command

The **show** command displays various system information.

Type:

```text
show
```

Press **TAB** twice to list all available subcommands.

---

## Example: Support Access

Run:

```bash
show support-access
```

This command displays whether Sophos Technical Support remote access is:

* Enabled
* Disabled

---

## Example: License Information

Run:

```bash
show log licenses
```

This displays licensing information for the firewall.

---

# ✅ CLI Navigation Tips

* Use **TAB** twice to display available commands.
* Use **TAB** for command auto-completion.
* Press **Ctrl + C** to stop running commands such as `ping` or `traceroute`.
* Press **0** to navigate back through menu levels.
