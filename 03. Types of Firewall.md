## Types of Firewall (Based on Implementation)

Firewalls can be classified **based on how and where they are implemented**.  
There are **three main types of firewalls**:

1.  **Hardware Firewall**
2.  **Software Firewall**
3.  **Virtual Firewall**

***

## 1️⃣ Hardware Firewall

### What is a Hardware Firewall?

A **hardware firewall** is a **physical device** installed between the **internal network (LAN)** and the **internet (WAN)**. It protects all devices behind it.

👉 Examples:

*   Sophos XGS Series
*   FortiGate Hardware Firewall
*   Cisco ASA / Firepower

***

### How It Works

    Internet ⇄ Hardware Firewall ⇄ Internal Network

All traffic entering or leaving the network must pass through the firewall.

***

### Real Scenarios

✅ **Scenario 1: Office Network Protection**  
An organization installs a Sophos XGS firewall at the network gateway.  
All employee PCs, servers, and printers are protected without installing security on every device.

✅ **Scenario 2: Perimeter Security**  
The hardware firewall blocks unauthorized WAN → LAN access attempts from hackers.

✅ **Scenario 3: High Performance Requirement**  
A company with heavy traffic (ERP, VPN, VoIP) uses hardware firewall for better throughput and stability.

***

### Advantages

*   High performance
*   Protects entire network
*   Centralized management
*   Ideal for enterprises & SMBs

### Disadvantages

*   Higher cost
*   Requires physical space and power

***

## 2️⃣ Software Firewall

### What is a Software Firewall?

A **software firewall** is installed **directly on a computer or server**.  
It protects only the device it is installed on.

👉 Examples:

*   Windows Defender Firewall
*   Sophos Endpoint Firewall
*   Linux iptables / UFW

***

### How It Works

    Application ⇄ Software Firewall ⇄ Network

The firewall controls inbound and outbound traffic per application or port.

***

### Real Scenarios

✅ **Scenario 1: Personal Laptop Security**  
A user enables Windows Firewall to block unauthorized connections while using public Wi‑Fi.

✅ **Scenario 2: Server-Level Protection**  
A Linux server uses iptables to allow only SSH and HTTP traffic.

✅ **Scenario 3: Additional Layer of Security**  
Endpoints inside a network still have host-based firewall for defense-in-depth.

***

### Advantages

*   Low cost / free
*   Easy to configure
*   Device-level protection

### Disadvantages

*   Protects only one system
*   Depends on system performance
*   Not suitable as the only firewall for enterprises

***

## 3️⃣ Virtual Firewall

### What is a Virtual Firewall?

A **virtual firewall** runs as a **virtual machine or cloud-based appliance**.  
It provides firewall functionality in **virtualized or cloud environments**.

👉 Examples:

*   Sophos Firewall Virtual
*   AWS Network Firewall
*   Azure Firewall
*   VMware NSX Firewall

***

### How It Works

    Cloud / Virtual Network ⇄ Virtual Firewall ⇄ Virtual Machines

***

### Real Scenarios

✅ **Scenario 1: Cloud Infrastructure Security**  
A company hosting servers on AWS deploys a virtual Sophos Firewall to protect cloud workloads.

✅ **Scenario 2: Data Center Virtualization**  
In a VMware environment, a virtual firewall controls traffic between virtual servers.

✅ **Scenario 3: Hybrid Network**  
Organization connects on‑prem firewall with cloud virtual firewall via site‑to‑site VPN.

***

### Advantages

*   No hardware needed
*   Scalable and flexible
*   Ideal for cloud and virtual environments

### Disadvantages

*   Depends on hypervisor/cloud performance
*   Requires virtualization knowledge

***

## 🔁 Comparison Summary

| Type              | Deployment         | Best For                     |
| ----------------- | ------------------ | ---------------------------- |
| Hardware Firewall | Physical appliance | Offices, enterprises         |
| Software Firewall | Individual system  | PCs, servers                 |
| Virtual Firewall  | VM / Cloud         | Cloud & virtual data centers |

***

## ✅ Final Summary

> **Hardware firewalls protect networks, software firewalls protect individual systems, and virtual firewalls protect cloud & virtual environments.**
